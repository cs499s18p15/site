Conclusions

Changes to Design

When we first started this project, our requirement was to create a minimum viable product for a wallchart tool. Essentially, our customer wanted a tool that would allow users to drag sticky notes between columns, and allow other users to see those changes in realtime. Fortunately, we were able to accomplish this relatively quickly! As we made steady progress, our customer started to see opportunities to request new features that would improve the usability of the wallchart tool. We kept in constant contact with our product owner, Troy, and worked to prioritize these new feature requests as we steadily made improvements to the project.

Lessons Learned

It's been a very rewarding experience putting together this web app for Toyota. We've learned how to communicate with our customer, and manage their expectations with what we could conceivably complete in our small time frame. We've also all learned a lot about various web technologies and how they link together, for example connecting to and querying a mysql database from a node server.
