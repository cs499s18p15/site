Screens


Welcome Page


The Welcome page is the first page the user will see. It has the login and register options. It is also the page users will see when they first login. It will be a little different when the user logs in to show that they were succesful. This page is our first interaction with the user.


Login Page


The login page lets the user enter their username and password and then checks the authenication. If the user exists and the hashed password is correct to our stored copy then we allow the user access to the site. There are 3 different levels of user, read-only, read-write, and admin.


Register Page


The registration page is where new users can come to create an account. As of right now a new user has read permission, but does not have write permission. They will get write permission once the admin approves their account and they are an authenticated user. In the future we will include email verification and password recovery.


Wallchart Page


The wallchart page is where team members and managers can view project status and add comments to projects. As of right now we have implemented the ability to drag projects into different categories. We are working to update the project to enable new features for the users. There are three columns, X, Triangle, and Circle. Circle is good, triangle is okay, and X is bad. Our customers want to be able to view all projects on the screen and access all projects' status in 7 seconds or less. This view will be the primary means for managers and team members to communicate project status and any pain points.


Completed Page


The Completed Page is used to show users projects from the past that have been completed. Each project includes a start time stamp and an end time stamp. It sorts the completed projects by most recent first. Team members are able to view completed projects, as well as delete projects and revert projects back to the wallchart page if needed. The Completed Page gives the user a nice overview of past project work.


Admin Page


The admin page is used for authenicating new users. Without approval from the admin, the user can only have read access. Once the admin approves the user they can then modify projects, add projects, and use the wallchart tool to the exent of their user access. The admin page is the first means of approving users and controlling security access to the teams' data and project status. Some users will not be Toyota Team Members and will not need write access.
