Testing


Unit Testing

  • Check project move route to ensure it updates database properly.
  • Check project add/delete routes to ensure the database is updated.
  • Check edit project route to ensure the database is updated.
  • Check the user routes to ensure the database is updated.

Function/Integration Testing

  • Drag 'sticky note' to a column and create a new project, then check if the new project was added correctly to the database.
  • Modify project information via project inspector and submit, then check to see if changes are reflected properly in the database.
  • Move a project from one column to a new column and check if the column was updated in the database.
  • Register a new user then check to see if the database has the new user added.
  • As an admin, authenticate a user then check to see if this is reflected in the database and also that the new user has the correct permissions inside the web app.
  • Ensure instructions and help menus are consistent.

Customer/User Testing

  • Test the application with as many users as possible simultaneously and see if there is any lag in real time updates.
  • Test the application with as many users as possible simultaenously in order to see how the site handles stress and crashing.
  • Ensure that users are able to update/move the projects that correspond to their assigned team.
  • Ensure that users feel as there are sufficient instructions provided and that the web application is user friendly.

System Testing

  • Test the application thoroughly with the required browsers: Microsoft Edge, Internet Explorer, Google Chrome. Incidentally, the application works very well with Firefox also.
  • Test the application with the Surface Hub at Toyota to ensure that it displays properly
  • Test the application with the Surface Hub at Toyota to ensure that the touch screen works well for moving projects and that they are properly updated in the database.
  • Test to see what the web page looks like when printed out.
  • Test the application on mobile phones (multiple iPhone and Android versions) as well as tablets in a variety of versions of the operating systems. This is so we are able to list what mobile operating systems are supported as of this time.
  • Check for obvious security problems (i.e. get some people to try to hack in to see if they are able to obtain sensitve information via a variety of methods).
