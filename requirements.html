Requirements


Requirements in priority order

  • Interactive "wallchart" front-end web application that pulls project information from a project database and displays it accurately. Users can create project elements of three separate project types and edit their information and column on the chart.

    (essential)


  • The front-end sends user submitted changes to back-end which then updates the database and changes the information displayed to all users. The database will include all information about each project including the title, description, stage/status, etc.

    (essential)


  • Users can create accounts with display names to edit projects. The application should show who updated what.

    (essential)


  • User authentication: Users have accounts and are authorized to edit only the projects to which they are a part of. Management will be able to create administrator accounts and are allowed to edit projects based on their title.

    (stretch)


Effort and Size Estimation

This project will consist of two primary "modules"-- a node.js server on the back-end and a front-end implemented primarily using the Vue Javascript library and we will include the routes in this estimation. There will also be a NoSQL database using MongoDB, but we cannot estimate the size the database will potentially reach. As for effort, making the server do real-time updates on a project's column movement on all users' screens will be the last feature implemented and potentially the most difficult, meaning it will require the most effort. The rest of the project will need to be completed first to ensure that these features work.

As far as size estimation (in lines of code) goes, we can estimate these numbers as follows (please note these estimations are generous and somewhat arbitrary):

  • The server will likely be less than 500 lines of code.
  • The front-end (in total) will likely be between 2500 - 3000 lines of code.

Platforms, Tools, and Languages

Summary:

Wallchart Tool is being built as a Single Page Application (SPA) with a MEVN stack (MongoDB, Express, Vue.js, Node.js). In the front-end, the user interface will be built with Vue.js. In the back-end, a Node server will not only serve the Vue app for users to access with a web browser, but also host an Express API. When the Vue app makes any requests to the server via this API, we can apply any necessary CRUD operations to the data we are storing in MongoDB. MongoDB is a NoSQL database service, which allows us to store and retrieve items without needing to specify a schema.

To package our web app, we are using Browserify. This allows us to use any of countless available javascript modules installed via npm (Node package manager).

Packages Used:
  • Axios: to facilitate sending requests from the front-end to the back-end
  • Socket.io: to implement real-time updates on the page as changes are made to the server. This package will allow the server to emit events when data is changed that the front-end will catch and handle appropriately.
  • Jsonwebtoken: to handle user authentication and eliminate the need for sessions.

Languages:

Because we are using Node.js for our server and Vue.js to create the interface for our web app, we can use Javascript for the front-end and back-end. Handy!
